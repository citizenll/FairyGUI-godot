# FairyGUI Godot 强类型绑定代码生成规范

状态：已实施  
目标版本：0.2  
适用范围：Godot 4.x，主要在 Godot 4.7 上验证

## 1. 目标

本功能用于消除业务代码中的 FairyGUI 名称字典和重复 `get_child()`、`get_controller()`、`get_transition()` 调用，并提供接近 Godot `@onready` 的使用体验。

目标业务代码：

```gdscript
@onready var ui: UI_EquipMain = %EquipView.fairy


func _ready() -> void:
	ui.tab_list.on(FGUIEvents.CLICK_ITEM, _on_tab_clicked)
	ui.ctrl.selected_index = 0
```

生成代码和业务代码必须分离。重新导入或重新生成不得覆盖业务脚本。

## 2. 非目标

- 不把 FairyGUI 动态子对象保存为 Godot 场景节点。
- 不生成指向预览节点的 `$NodePath`。
- 不默认自动连接业务事件处理函数。
- 不要求安装 FairyGUI Editor 插件；外部插件只能作为未来可选的元数据增强来源。
- 不在 `EditorImportPlugin._import()` 的工作线程内写入 GDScript 源文件。

## 3. 核心原则

1. 绑定始终按 FairyGUI 语义名称执行，不按子对象索引执行。
2. 默认忽略空名称和 `n<number>` 形式的自动子对象名称。
3. Controller 和 Transition 只要名称非空就生成。
4. 生成结果必须确定：相同输入产生字节一致的输出。
5. 生成器只能删除自己在上一次生成清单中记录且带自动生成标记的文件。
6. 扫描或生成失败时保留上一份可工作的生成结果。
7. 手工注册的 `FGUIObjectFactory` 扩展优先于自动生成扩展。
8. 生成注册表缺失或损坏时，未使用强类型绑定的 FairyGUI 功能必须继续工作。
9. 导出插件必须显式加入注册表和清单记录的生成脚本，使“仅导出场景依赖”模式也可使用动态注册表。

## 4. 运行时接口

### 4.1 FGUIComponent

组件完成子对象、Controller、Relation、Transition 和初始 Controller 状态构建后调用：

```gdscript
func _on_construct() -> void:
	pass
```

生成类覆盖该方法并绑定成员。

组件提供以下强校验方法：

```gdscript
func require_child(name: String, expected_type: Variant = null) -> FGUIObject
func optional_child(name: String, expected_type: Variant = null) -> FGUIObject
func require_controller(name: String) -> FGUIController
func require_transition(name: String) -> FGUITransition
```

缺失或类型错误必须输出包含 package、component、member 和 expected type 的结构化错误。

`optional_child()` 用于 branch 特有成员：缺失时返回 `null`，存在但类型错误时仍输出结构化错误。

### 4.2 FGUIView

`FGUIView` 增加：

```gdscript
signal fairy_ready(value: FGUIObject)

@export var component_script: Script
var fairy: FGUIObject
```

运行时必须在自身 `_ready()` 内同步创建 FairyGUI 对象，使父节点的 `@onready` 和 `_ready()` 可以读取 `fairy`。编辑器预览继续使用延迟刷新，避免 Inspector 属性连续变化导致重复构建。

`component_script` 是显式覆盖项；未设置时使用自动生成注册表解析出的组件脚本。

### 4.3 FGUIObjectFactory

对象工厂维护两组扩展：

- 手工扩展：现有 `set_extension()` API 注册，优先级最高。
- 自动生成扩展：从生成注册表加载，可整体替换和重新加载。

默认注册表路径：

```text
res://generated/fairygui/registry.gd
```

可通过 `fairygui/codegen/registry_path` 项目设置覆盖。注册表不存在或执行失败时回退到普通 FairyGUI 类型。

注册表只保存 URL 到脚本资源路径的映射，组件首次创建时再加载对应脚本。这样编辑器预览和运行时启动不会一次性解析所有生成类，也避免注册表在编辑器文件扫描阶段因批量 `preload` 暂时不可实例化。

编辑器内的 `FGUIView` 预览只解析和渲染 `.fui` 数据，不自动执行生成类或业务脚本；运行项目后由注册表启用强类型组件。显式传入可在编辑器执行的脚本仍由调用方自行负责其 `@tool` 安全性。

## 5. 生成触发

`.fui` 导入器只负责生成 `FGUIPackageResource`。编辑器插件监听 `EditorFileSystem.resources_reimported`，过滤 `.fui` 后在主线程延迟执行一次合并生成。

触发方式：

- `.fui` 导入或重新导入后的自动生成。
- Godot “项目 > 工具”菜单中的 Generate FairyGUI Bindings。
- `FGUIPackageResource` 和 `FGUIView` Inspector 中的生成按钮。

自动生成由 `fairygui/codegen/auto_generate` 控制，默认开启。插件自身示例工程可以显式关闭。

## 6. 扫描模型

生成器加载所有启用的 `.fui` 资源及其同目录依赖包，在内存中建立统一 Schema：

```text
Package
  source_path
  id
  name
  components[]
    id
    name
    exported
    object_type
    children[]: source_name, source_item, object_type, order
    controllers[]: source_name, order
    transitions[]: source_name, order
```

扫描直接读取组件 `raw_data`，不得通过实例化 UI 节点获取成员，以避免旧绑定脚本、音频、网络 Loader 或用户扩展产生副作用。

默认只为 FairyGUI Editor 标记为 Exported 的公开组件生成业务类。未导出的内部组件继续使用对应 `FGUI*` 基类，避免为皮肤、滚动条和纯内部装饰组件产生无意义代码。需要直接操作列表 Item 等内部组件时，可以显式开启 `include_internal_components`。

依赖包即使关闭代码生成也可以参与类型解析，但不得为其输出生成类；其内容哈希仍进入清单，依赖变化会触发调用方组件重新计算。

组件的 branch 和高分辨率变体合并到公开基础组件的同一个 Schema，不为内部变体单独生成业务类。所有变体都存在的成员使用强校验绑定；仅部分变体存在的成员为可空绑定，不在缺失分支输出错误。

## 7. 命名规则

- 组件类：`<class_prefix><PackagePascal><ComponentPascal>`。
- 默认 `class_prefix`：`UI_`。
- 成员：FairyGUI 原名转换为 `snake_case`。
- 非 ASCII 或无法转换的名称使用稳定的 package/item ID 回退名。
- GDScript 保留字添加 `ui_` 前缀。
- 同一作用域内转换后重名时追加 `_2`、`_3`，同时产生警告。
- 原始 FairyGUI 名称始终保存在查找字符串中，重命名变量不会改变运行时查找目标。
- 成员按类别和语义名称稳定排序；非 ASCII 回退名使用原名的稳定哈希，不依赖子对象索引。

## 8. 类型规则

- FairyGUI 内置 object type 映射到对应 `FGUI*` 类。
- 当前生成集合内的非循环组件引用通过显式脚本 `preload` 映射到对应生成类，不依赖编辑器全局类缓存的扫描顺序。
- 自引用或循环组件引用回退到对应内置基类，避免循环 `preload` 导致注册表无法加载，并产生生成警告。
- 同名成员在不同变体中类型不一致时回退到共同 `FGUIComponent` 或 `FGUIObject` 基类，并产生生成警告。
- 无法解析的跨包组件引用回退到其内置基类或 `FGUIComponent`，不得生成无法加载的类型名。
- Controller 使用 `FGUIController`。
- Transition 使用 `FGUITransition`。

## 9. 输出结构

默认输出：

```text
res://generated/fairygui/
  registry.gd
  manifest.json
  equip/
    ui_equip_main.gd
    ui_equip_list_item.gd
```

每个生成文件首行包含固定标记：

```text
# Generated by FairyGUI Godot. DO NOT EDIT.
```

`manifest.json` 记录生成器版本、生成选项、输入哈希、生成文件和 URL 到脚本的映射。输出先写临时文件，再替换目标文件；内容未变化时不改写。

替换前备份本轮涉及的旧文件，全部脚本加载校验通过后才清理备份。进程内任一步失败时回滚本轮已安装文件；下次生成会优先恢复残留备份和临时文件。

## 10. 生成类契约

每个组件生成类必须包含：

- `class_name`。
- 正确的 `FGUI*` 基类。
- `URL`、`PACKAGE_ID`、`PACKAGE_NAME`、`COMPONENT_ID`、`COMPONENT_NAME` 常量。
- 强类型成员变量。
- `_on_construct()` 名称绑定。
- `create_instance(package: FGUIPackage = null)` 便捷创建方法。

生成类不得包含业务事件、网络请求或状态逻辑。

## 11. 编辑器体验

Inspector 操作必须提供：

- Generate Bindings：重新生成项目绑定。
- Open Binding：打开当前 `FGUIView` 对应的生成脚本；未生成时禁用或先生成。

生成完成后请求 EditorFileSystem 扫描，并使 `FGUIView` 刷新预览。错误必须显示在 Godot 输出面板，汇总中包含生成、跳过、删除和警告数量。

## 12. 故障处理

- 单个包损坏：本轮不修改任何生成文件，输出错误。
- 注册表缺失：使用基础 FGUI 类型继续创建 UI。
- 某个成员缺失：该字段为 `null`，输出一次结构化错误，不中止其余成员绑定。
- 类型不匹配：字段为 `null`，输出预期类型和实际类型。
- 关闭全局 `auto_generate`：不自动触发生成或删除；手工生成仍按各 `.fui` 的 `codegen/enabled` 导入选项更新输出。
- 依赖包缺失：相关成员回退到基础类型并输出警告。

## 13. 验收标准

1. `.fui` 重排子对象但名称不变，生成绑定和运行结果不变。
2. 业务脚本无需名称字典或成员级 `get_child()`。
3. `FGUIView.fairy` 在父节点 `_ready()` 时可用。
4. 显式 `component_script`、手工 factory 扩展和自动扩展优先级正确。
5. 生成两次且输入不变时，文件内容及时间戳不变化。
6. 删除组件后只删除对应自动生成文件，不删除用户文件。
7. 注册表缺失、语法错误或加载失败不会阻止普通 FGUIComponent 创建。
8. Godot 4.7 下通过静态检查、生成器测试、运行时绑定测试、编辑器导入测试和现有 smoke tests。
9. 使用“仅导出所选场景及依赖”时，生成注册表和所有绑定脚本仍进入导出包。
10. 切换 FairyGUI branch 后，公共成员保持强类型，分支特有成员允许为 `null` 且不产生缺失绑定错误。
